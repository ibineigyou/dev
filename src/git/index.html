SETUP

環境構築ガイド

ステップバイステップで開発環境を整えましょう

1

Gitのインストール

Terminal
# Windows: Git for Windowsをダウンロード https://git-scm.com/download/win # macOS brew install git # Ubuntu/Debian sudo apt install git
⬇️ 公式ダウンロード
2

Eclipse IDEのインストール

手順
1. eclipse.org にアクセス 2. "Eclipse IDE for Java Developers" を選択 3. OSに合わせてダウンロード 4. インストーラーを実行
💡

EGit（Git連携プラグイン）は標準搭載されています

3

Gitの初期設定

Terminal
# ユーザー名の設定 git config --global user.name "Your Name" # メールアドレスの設定 git config --global user.email "your@email.com" # デフォルトブランチ名をmainに git config --global init.defaultBranch main
4

SSH鍵の設定

Terminal
# SSH鍵の生成 ssh-keygen -t ed25519 -C "your@email.com" # 公開鍵の表示（GitHubに登録） cat ~/.ssh/id_ed25519.pub # 接続テスト ssh -T git@github.com
HANDS-ON

実践演習

実際に手を動かしながらGit操作をマスターしましょう

📁 Exercise 1: 初めてのコミット

1

新規Javaプロジェクトの作成

Eclipseで練習用のJavaプロジェクトを作成します。

File → New → Java Project Project name: GitPractice → Finish
2

Gitリポジトリの初期化

プロジェクトをGitで管理するように設定します。

プロジェクトを右クリック → Team → Share Project → Git を選択 → Create Repository → Finish
3

ステージングとコミット

変更をGitに記録します。

Team → Commit Commit Message: "Initial commit" → Commit
✅

完了チェック

プロジェクト名に [GitPractice main] と表示されていれば成功です

🌿 Exercise 2: ブランチを使った開発

1

新規ブランチの作成

Team → Switch To → New Branch Branch name: feature/greeting → Finish
2

機能を追加してコミット

// 新しいメソッドを追加 public static void greet(String name) { System.out.println("Welcome, " + name); }
3

mainにマージ

Team → Switch To → main Team → Merge → feature/greeting → Merge

☁️ Exercise 3: GitHubへのプッシュ

1

GitHubでリポジトリを作成

github.com → + → New repository Repository name: GitPractice → Create repository
2

リモートを登録してプッシュ

Git Repositories View → Remotes → Create Remote URI: git@github.com:USER/GitPractice.git Team → Push to Upstream

初学者がつまずきやすいポイントをQ&A形式で解説

Gitはローカルで動作する分散型バージョン管理システム。GitHubはGitリポジトリをホスティングするWebサービス。Gitは「ツール」、GitHubは「サービス」です。
1行目に変更内容を50文字以内で要約。例：「Add user authentication」「Fix null pointer exception」
リモートにローカルにない変更がある場合に発生。git pullで変更を取り込み、コンフリクトがあれば解決後に再度push。
Eclipseでは競合ファイルを右クリック → Team → Merge Tool で3ペイン表示で編集。必要な変更を反映して保存、コミット。
Eclipse + Javaでは: .project, .classpath, .settings/, bin/, *.class, target/ など。
